Writing my first Tezos contract

'Too many options! What do I choose?' - Yes that was me. But you don't need to be. Learn how you can build your first Tezos DApp!

Date Published: 2022-02-03T11:07:14+05:30

Why did I write this article?

Too many options! What do I choose?

~ A confused Tezos DApp enthusiast

Yes that was me.

So I wanted to get my feet wet with Tezos - since quite sometime. Infact, joining the local Tezos Developers Community only boosted my eagerness to work with blockchain in general, and Tezos in particular.

Note

If you don't know what Tezos is, you should definitely read my article on Tezos ! ðŸ˜ƒ

But getting started with Tezos is, to say the least, daunting at first, given the sheer number of options people have while building on it. You can build on top of the Tezos blockchain using either one, or a combination of the following languages:

  • JavaScript/TypeScript : The official documentation uses it.
  • Python : The Cryptoverse Wars course uses it.
  • Michelson : The native language for the Tezos blockchain
  • OCaml : Used by the folks at Zastrin for their Tezos blockchain course.
  • Any of the LIGO Languages ( JsLIGO , CameLIGO , PascalLIGO , ReasonLIGO )

And all this is discounting the number of options you have at hand to add into your project. Which may be a good thing for experienced developers, but an ABSOLUTE NIGHTMARE for people trying to get started with it.

Thankfully, helvantine has written a good article on Tezos on making your first Tezos contract (which helped me out to make my first contract too), and I'll be using that as a reference for this post.

That said, I actually noticed a lack of details in the article - the inner workings not explained lucidly (perhaps the author expected the audience with a background of software dev in general?), and I am trying to simplify some of the seemingly arcane stuff for everyone to understand. ðŸ˜ƒ

Setting up the environment

As in every blog, I'll say this one again: Setting up your development environment is one of the biggest factors between a clean project that's developed quick, and a messy project that takes days to debug and test.

Even Sun Tzu agrees with me:

(Original content made by Yours Truly ðŸ˜Ž)

So without any further ado, let's set up our development environment for developing on Tezos!

Step 0: Evaluate what you'll need for the project

Yes, this is an often-overlooked but really important part of setting up your environment.

Why, you may ask. Consider the following scenarios:

  • You have a small project. Installing unnecessary tools bloats your work and makes it difficult for you to manage the size of the project.
  • Often projects have dependencies that conflict. This is an even greater problem that's addressed in the next step.

For this project, we'll be needing:

  • SmartPy CLI
  • Python 3 (I have tested with 3.9.5)
  • NodeJS (I have tested with v16.13.2, the latest LTS version)
  • An sh shell (like Bash or Zsh) (I use zsh )
  • Temple Wallet Web Browser extension (I have tested on Google Chrome)
  • Telegram App with Telegram Account.

Quick Tip

You should use Node Version Manager (NVM) for maintaining your NodeJS environment versions. Install it using: 

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash
To use the latest LTS version in your project, go to your project root directory in your terminal and type: 
nvm install --lts
nvm use --lts


Step 1: Create your project with the proper environment and ONLY the necessary tools

First, we'll create a directory called First Contract and enter into it: 

mkdir "First Contract"
cd "First Contract"

The folder First Contract is your project root directory . Inside First Contract , we'll create some more directories: 

mkdir builds contracts compilation deployment

Setting up SmartPy CLI

According to the SmartPy Blog:

SmartPy is a high-level smart contracts library and comes with related tools in the form of SmartPy.io to greatly ease the accessibility, understandability and provability of smart contracts on Tezos. It combines several popular aspects of software engineering to create an attractive language for smart contract development on Tezos.

And according to their official website:

The SmartPy language is available through a Python library for building and analyzing Tezos smart contracts. It comes with various tools: a compiler that generates Michelson code, a simulation engine, a contract explorer, etc.

To install SmartPy CLI, type the following in your terminal: 

sh <(curl -s https://smartpy.io/cli/install.sh)

This should install SmartPy CLI on your home directory, specifically at ~/smartpy-cli/ .

To verify the installation, type the following in your terminal: 

~/smartpy-cli/SmarPy.sh --version


Quick Tip

Typing ``~/smartpy-cli/SmarPy.sh`` for every little action you do is a drain on productivity, especially over prolonged periods. To avoid this, use this hack: - Type the following in your terminal: 

nano ~/.bashrc
if you are using bash , or: 
nano ~/.zshrc
if you are using zsh shell. - Copy the following into the text editor window that opens: 
alias smartpy="~/smartpy-cli/SmartPy.sh"
Press ``Ctrl + S`` to save the file, and ``Ctrl + X`` to exit. - Close the console window(s), and open them again. Now doing ``smartpy --version`` should yield the same output as ``~/smartpy-cli/SmartPy.sh --version``.


Setting up Python

We need to install the Python virtualenv package to create our Python virtual environment. Install and configure it for the project with: 

pip install virtualenv
python3 -m virtualenv env
source env/bin/activate

There will be a few lines of output, mostly telling which version of Python is being used to make the virtual environment. Also, you'll most probably notice some extra text on your terminal cursor, possibly showing env somewhere (on the left for Windows CMD or PowerShell , left for bash users, and either left or right for zsh users).

What this does: This basically installs virtualenv package for Python, then creates a virtual environment for the project, and then activates it.

Installing Python packages

For the project, we'll be needing the following Python packages:

  • future (I've used v0.18.2)
  • numpy (I've used v1.22.1)
  • scipy (I've used v1.7.3)
  • smartpy (I've used v0.2.1)

Install it using: 

pip install future numpy scipy smartpy

Or alternatively, you can download the requirements.txt file inside your project root and from the terminal at the project root, type: 

pip install -r requirements.txt

in the terminal.

Setting up NodeJS and NPM

Now, we need to set up the node environment for the project. We do it by: 

nvm install --lts
nvm use --lts
npm init --yes

What this does: This uses the latest LTS version of NodeJS for your project, and initializes your NPM environment using default settings.

Installing Node dependencies from npm

We'll also need the following npm packages:

  • ts-node (I've used v10.4.0)
  • dotenv (I've used v15.0.0)

We install them using:

Understanding the workflow

The entire workflow is like this:

  • The contracts/ folder contains the Tezos smart contracts written in Python with SmartPy.
  • The .env file contains the environment variables to deploy the contracts to. For this project, it's your wallet private key ORIGINATOR_PRIVATE_KEY and the Tezos RPC URL TEZOS_RPC_URL .
  • The deployment/ folder will contain deploy.ts , a script that will deploy the compiled files to the Tezos Test Network (TestNet) specified by the variable TEZOS_RPC_URL in thr .env file.
  • The compile.sh file will first compile the contracts in the contracts/ folder one-by-one and save them in a temporary hidden folder called .smartpy_out/ in the project root directory. From there, it will run deployment/deploy.ts so that the code is deployed according to the environment variable settings in the .env file.

We're finally good to go! Give yourself a pat on the back if you have made it this far! We'll be starting the coding part! ðŸ˜ƒ

Coding the required scripts (while understanding them)

Coding compile.sh 

#!/usr/bin/env bash

set -e -o pipefail

echo "----------------------------------------"
echo "Compiling Dao Contracts"
echo "----------------------------------------"

# Expected location of SmartPy CLI.
SMART_PY_CLI=~/smartpy-cli/SmartPy.sh

# Output directory
OUT_DIR=./compilation

# Array of files to compile.
CONTRACTS_ARRAY=($(cd contracts && ls));

echo "----------------------------------------"
echo "Contracts found:"
echo $CONTRACTS_ARRAY
echo "----------------------------------------"

# Ensure we have a SmartPy binary.
if [ ! -f "$SMART_PY_CLI" ]; then
    echo "Fatal: Please install SmartPy CLI at $SMART_PY_CLI" && exit
fi

# Compile a contract.
# Args <contract name, ex: minter> <invocation, ex: MinterContract()> <out dir>
function processContract {
    CONTRACT_NAME=$1
    OUT_DIR=$2
    CONTRACT_IN="${CONTRACT_NAME}.py"
    CONTRACT_OUT="${CONTRACT_NAME}.tz"
    CONTRACT_COMPILED="${CONTRACT_NAME}/step_000_cont_0_contract.tz"

    echo ">> Processing ${CONTRACT_NAME}"

    # Ensure file exists.
    if [ ! -f "$CONTRACT_IN" ]; then
        echo "Fatal: $CONTRACT_IN not found. Running from wrong dir?" && exit
    fi

    # Test
    echo ">>> [1 / 3] Testing ${CONTRACT_NAME} "
    $SMART_PY_CLI test $CONTRACT_IN $OUT_DIR
    echo ">>> Done"

    echo ">>> [2 / 3] Compiling ${CONTRACT_NAME}"
    $SMART_PY_CLI compile $CONTRACT_IN $OUT_DIR
    echo ">>> Done."

    echo ">>> [3 / 3] Copying Artifacts"
    cp $OUT_DIR/$CONTRACT_COMPILED $CONTRACT_OUT
    echo ">>> Written to ${CONTRACT_OUT}"
}

echo "> [1 / 3] Unit Testing and Compiling Contracts."
for i in ${!CONTRACTS_ARRAY[@]}; do
    echo ">> [$((i + 1)) / ${#CONTRACTS_ARRAY[@]}] Processing ${CONTRACTS_ARRAY[$i]}"
    processContract ${CONTRACTS_ARRAY[$i]} $OUT_DIR
    echo ">> Done."
    echo ""
done
echo "> Compilation Complete."
echo ""

# Remove other artifacts to reduce noise.
echo "> [2 / 2] Cleaning up"
rm -rf $OUT_DIR
mkdir $OUT_DIR
echo "> All tidied up."
echo ""

echo "----------------------------------------"
echo "Task complete."
echo "----------------------------------------"
